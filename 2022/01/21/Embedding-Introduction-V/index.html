DeepWalk 的优化 - Node2Vec

可以这么理解，deepwalk 主要是基于 DFS 邻域的 graph embedding 算法，而 LINE 则是基于 BFS 邻域的 graph embedding 算法。接下来介绍的 node2vec 则是总和考虑 DFS 与 BFS 的 graph embedding 算法，是结合 DFS 与 BFS 的随机游走的 deep walk。

算法原理

定义：

  1. f(u)f(u)f(u) 为顶点 uuu 到其对应 embedding 向量的映射

  2. NS(u)N_S(u)NS​(u) 为在采样策略 SSS 下顶点 uuu 的近邻顶点集合

而 node2vec 优化的目标比较直接，就是希望找到最优映射 f(u)f(u)f(u)，使近邻顶点出现的概率最大化，即

maxf∑u∈Vlog⁡Pr(Ns(u)∣f(u))max_f\sum_{u \in V} \log Pr(N_s(u) | f(u)) maxf​u∈V∑​logPr(Ns​(u)∣f(u))

为了使该最优化问题可解，文章做了两个假设：

  1. 条件独立性假设。即在给定顶点 uuu 下，其近邻顶点出现的概率，与近邻集合中其他顶点无关。即

    Pr(Ns(u)∣f(u))=∏ni∈Ns(u)Pr(ni∣f(u))Pr(N_s(u) | f(u)) = \prod_{n_i\in N_s(u)} Pr(n_i | f(u)) Pr(Ns​(u)∣f(u))=ni​∈Ns​(u)∏​Pr(ni​∣f(u))

  2. 特征空间对称性假设。与 LINE 不同，假设一个顶点无论是作为原顶点还是近邻顶点，其共享同一套 embedding 向量。对应表达为

    Pr(ni∣f(u))=exp⁡(f(ni)⋅f(u))∑v∈Vexp⁡(f(v)⋅f(u))Pr(n_i | f(u)) = \frac{\exp(f(n_i)\cdot f(u))}{\sum_{v\in V} \exp(f(v)\cdot f(u))} Pr(ni​∣f(u))=∑v∈V​exp(f(v)⋅f(u))exp(f(ni​)⋅f(u))​

最终，根据假设条件，我们的最终目标函数表示为

maxf∑v∈V[−log⁡Zu+∑ni∈Ns(u)f(ni)⋅f(u)]max_f\sum_{v\in V}\left[-\log Z_u + \sum_{n_i\in N_s(u)} f(n_i) \cdot f(u)\right] maxf​v∈V∑​⎣⎡​−logZu​+ni​∈Ns​(u)∑​f(ni​)⋅f(u)⎦⎤​

其中 Zu=∑n∈Ns(u)exp⁡(f(ni)⋅f(u))Z_u = \sum_{n\in N_s(u)} \exp(f(n_i)\cdot f(u))Zu​=∑n∈Ns​(u)​exp(f(ni​)⋅f(u)) 的计算代价高，需要采用负采样技术进行优化。

采样策略

相比随机游走 deep walk 中的采样方式， node2vec 也是随机游走，但是是一种依概率随机游走，这种随机游走是有偏的。

比如给定当前顶点 vvv，访问下一个顶点 xxx 的概率我们设定为

P(ci=x∣c−1=v)=πvxZ, if (v,x)∈EP(c_i = x | c_{ - 1} = v) = \frac{\pi_{vx}}{Z} \text{, if } (v, x) \in E P(ci​=x∣c−1​=v)=Zπvx​​, if (v,x)∈E

P(ci=x∣ci−1=v)=0, otherwiseP(c_i = x | c_{i - 1} = v) = 0 \text{, otherwise} P(ci​=x∣ci−1​=v)=0, otherwise

其中 πvx\pi_{vx}πvx​ 表示顶点 vvv 与顶点 xxx 之间的转移概率（未 normalized）， ZZZ 则是归一化常数

文中 node2vec 引入两个参数 ppp 与 qqq 来控制随机游走的策略。

πvx=αpq(t,x)⋅wvx\pi_{vx} = \alpha_{pq}(t, x)\cdot w_{vx} πvx​=αpq​(t,x)⋅wvx​

其中 wvxw_{vx}wvx​ 表示顶点 vvv 与 xxx 之间的边权。而 αpq\alpha_{pq}αpq​ 则与上一次游走起点 ttt 和当次游走终点 xxx 有关，可知两个顶点的距离 dtxd_{tx}dtx​ 可以有三个取值：

  • dtx=0d_{tx} = 0dtx​=0，即两点属于同一个点，此时设定 αpq(t,x)=1p\alpha_{pq}(t, x) = \frac{1}{p}αpq​(t,x)=p1​，所以 ppp 又被称为 return parameter，表示马上重新访问原来 node 的可能性。
  • dtx=1d_{tx} = 1dtx​=1，即 t,xt, xt,x 与 vvv 三点均相邻，此时设定 αpq(t,x)=1\alpha_{pq}(t, x) = 1αpq​(t,x)=1
  • dtx=2d_{tx} = 2dtx​=2，即 xxx 远离 ttt 点，两点不直接相邻，此时设定 αpq(t,x)=1q\alpha_{pq}(t, x) = \frac{1}{q}αpq​(t,x)=q1​，所以 qqq 又被称为 in-out parameter。
